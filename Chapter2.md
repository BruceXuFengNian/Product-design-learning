# 第二章 从竞品中摸索业务
从标题中我们也能够看出来，这一章的主要内容就是研究竞品。简单地说，就是要产出一个竞品分析报告，这个分析报告要囊括谁用这个竞品做什么事情满足他的什么需求。我们在研究竞品的时候主要要整清楚下面这些问题：  
1. 使用客群：谁在用这个产品；
2. 使用场景：客群在什么情况下回使用这个产品；
3. 使用流程：客群在场景下怎么使用这个产品； 
4. 使用交互：产品通过什么交互模式让用户正确跑完整个使用流程；
5. 使用反馈：客群使用这个产品之后获得了什么；
6. 使用关系：产品的整个使用流程+反馈里有哪些角色相互作用。

前五条是让我们整体了解业务是怎么在产品里运行的，最后一条是了解整个产品在业务中的地位以及其为业务营造出来的生态。  

那么，为什么研究竞品，怎么研究竞品，以及怎么产出竞品分析报告，就是本章要向大家说清楚的内容。

## 为什么研究竞品

为什么研究竞品？这个问题有很多答案。

1. 了解现在行业的发展方向；
2. 了解竞争对手现在在做什么；
3. 了解竞争对手的解决方案，他们怎么解决用户场景的问题；
4. 了解竞争对手最近有什么创新。

这些答案最终其实都会指向一个结果——抄，还是不抄？如果是抄，怎么抄；如果不抄，为什么。

针对不抄，我们有很多情况，一般是目标客群不同，或者企业的发展不同，或者管理模式不同，市场占有率不同，商业化模式不同等等，这些在用户调研和市场调研以及产品商业化的工作中会有所涉及，我们这里就不过分说明这些内容，这个章节，我们就教大家做一件事：

抄，而且要抄出比原作者分数更高的作业。

## 竞品研究方法论

我们在研究竞品的过程中，主要是做下面这几个步骤：

1. 找竞品——哪些是我们的竞品？
2. 体验竞品——这些竞品是做什么的，这里面最强的、最有借鉴价值的竞品是哪个？
3. 总结——借不借鉴？怎么借鉴？

### 找竞品

回答第一个问题——找竞品。

#### 目标：找竞品来干什么

一般来说，找竞品来分析无非是要解决下面几个问题：

1. 产品定位：这个产品有没有市场，市场有多大，有没有切入的可能性；
2. 用户需求：什么用户会用这个产品，他用这个产品解决了他什么问题；
3. 业务流程：产品通过什么流程满足了用户的需求；
4. 产品设计：产品通过什么设计方式跑通了业务流程，包括页面设计、数据层设计和交互设计；
5. 运营方案：产品上线之后通过什么运营方式实现了销售或者推广。

我们在确认这次竞品研究的要点之后，再开始研究竞品，切勿上来什么都想研究，结果什么都没深入，没有深入研究的竞品报告是没有灵魂的。

#### 找到竞品的渠道

我们就要知道哪些地方能够找到竞品。

1. 新闻快讯
- 36氪：互联网最新资讯一般都在这里，我们可以从这里面了解到当下企业融资资讯，也可以知道当下有什么公司又发布了新产品，同时还有机会了解到一些免费可以参加的行业峰会。基本上很多意想不到的竞争者都是在这里放出他们的一手信息。
- i黑马：和36氪差不多，更多地偏向于报道创业企业发布的信息，还有一些产业新闻都可以在这里看到。

2. 产品经理论坛
- 人人都是产品经理：你想研究哪个领域，都可以跑进去看看有什么文章，这个里面有很多产品经理分享的产品研究或者设计思路文档；
- PMCAFF：高阶产品经理论坛，很多文章都比较有深度，大多都是分析业务或者产品设计思路的长文或问答；如果你要搜索什么竞品还需要注册，这点倒是比较麻烦。
- 知乎：不解释了，想看什么搜就是了，但是关键字要选好，别搜到一些乱七八糟的回答就行；比如你想研究IM产品 你就搜“IM软件 产品调研” 就行，词汇越专业约好，可以有效避免水军。

3. 应用市场
- 产品排行榜：如酷传，Appannie, 鸟哥ASO，易观等；appannie一般用来查ios的应用，酷传一般拿来查Android的应用，鸟哥和易观都有，不过部分内容需要付费；
- isv服务榜：这个主要针对b端服务，企业微信、钉钉服务市场都有这个排名。

通过这些渠道，我们能够了解视野当中有哪些产品，接下来还要去选和我们自己定位相似的产品，选定为竞品进行研究。



#### 确定研究的范围

选定这些竞品一般通过目标来确定竞品研究的范围

1. 产品定位：如果我们研究的是产品定位，那么就要找核心业务和客户群体和我们当前高度重合的产品；比如我们要做业务财务，那么我们上来就应该去找当前市场也在做业务财务的公司以及业务财务的行业报告；
2. 用户需求：如果我们研究的是用户需求，那么就要找业务逻辑或用户场景高度重合的产品；比如用户的目标是买菜，那么我们就应该去找满足用户买菜需求的各大app，比如盒马生鲜，叮咚买菜，每日优鲜等等；
3. 业务流程：如果我们研究的是业务流程，就应当直接把研究范围铺开，只要通过流程满足了某个特定方面的用户需求都可以是我们的研究对象；比如我们要研究下单流程，那么电商下单，外卖下单，会员下单，乃至游戏充值都可以是研究的对象；
4. 产品设计：如果我们研究的是产品设计，则要在业务流程的基础上再去细化，只要满足了类似业务流程的跳转逻辑的，都可以作为研究对象；比如用户下单的功能，电商平台会怎么做，外卖平台会怎么做，游戏充值又会怎么做，他们的页面是什么，他们的交互如何，针对同一个流程，他们有什么不一样的地方，字段、筛选和展示层面的东西都可以深入研究；
5. 运营方案：如果我们研究的是运营方案，就要稍微跳出一下产品功能层面的东西，回到产品定位和用户需求这个层面，针对同一个用户群体他们是如何运营促活以及转化的，得到和樊登读书会他们是怎么吸纳用户的，饿了么和美团他们是怎么去抢下沉市场的，拼多多和淘宝他们是怎么是怎么抢客户的。



### 体验竞品

通过我们确定的竞品研究范围，我们需要有针对性地去体验竞品。

从我们刚刚说到的5个范围，我们体验竞品的技能有以下几个点需要掌握：

1. 赛道判定：通过行业报告获取行业当前现状的信息
2. 选手调查：调查单一竞品的产品迭代周期和用户数据表现；
3. 运营调研：调查单一竞品的运营事件以及运营发力点和产品发力点之间的配合模式；
4. 产品研究：调查单一竞品的产品商业逻辑、业务逻辑和所支撑的业务及其业务流程；
5. 交互研究：调查单个业务流程下的产品使用逻辑、前端界面交互。



### 竞品间对比

体验竞品的时候会对比不同竞品之间在用户定位、运营策略、业务逻辑、产品交互的差别，以此来判断自己如何取长补短。

1. 对比产品版本迭代路径

   一般来说，产品迭代路径可以很清晰地看出来一个产品的生命周期，同时也可以发现一个产品逐渐进入成熟的阶段是通过什么方式实现了进一步的增长；当然核心的目标还是了解产品生命周期以及迭代过程与运营的配合模式；

2. 对比产品商业模式/业务逻辑

   我们体验产品的时候，去思考为什么如此设计之前，要基于面向的客户和产品与客户的交互逻辑来比对产品的设计思路。所以在研究设计之前，要先比对产品的商业模式和业务逻辑；

3. 对比产品结构/用户交互

   进入产品体验，首先要了解所有功能在产品里的排布，同时要了解用户怎么通过各个功能完成业务流程；这个过程能够加强我们对人机交互的理解能力；

4. 对比用户场景/界面交互

   了解业务流程之后，再针对用户的实际使用场景去重点对比界面元素和界面上的交互方式；这个过程能够让我们站在用户的角度仔细体会产品是怎么满足用户需求的，它是怎么让用户看懂的，又是怎么引导用户完成操作的。



## 竞品报告怎么写

竞品报告就是我们把整个找的过程和体验的过程记录下来的文档。包括这两个部分在内，我们还需要再多回答一个问题：我们要不要抄，抄的话怎么抄。

针对这个问题，我们首先就要弄清楚我们自己。

1. 我们打算做什么；
2. 我们想满足哪些用户的哪些场景。

弄清楚我们自己之后，我们再来把对竞争对手的信息叠加上来：

1. 我们研究的这些竞品，他们分别是怎么解决的；
2.  针对我们需要满足的场景，哪些是可以借鉴的；
3. **我们怎么设计，可以基于竞品的基础上再做得更好。**

大家应该看出来了，我们写到最后，核心的目标就是回答最后一个问题——我们怎么做得更好。

这一条就放在第三章里面和大家说了。一般情况下，如果不经过踏实的产品设计训练，很有可能会东抄抄西抄抄抄出一个四不像或者大杂烩，我们要做的，就是杜绝这种情况的发生。

### PART1：外部环境调研

 第一步部分对应的是我们找竞品的部分。

调研外部环境的部分，按照我们对竞品所在产业，竞品本身的公司情况进行调研。根据前半部分给到的资料，这个地方我们可以从以下几个地方获取内容:

1. 找竞品:通过新闻、论坛、应用市场等多个渠道找竞品；
2. 找行业报告：艾瑞咨询的行业报告<https://report.iresearch.cn/>

内容准备完了之后，我们在part1里面回答两个问题。

1. 当前的外部环境如何（行业+企业）；
2. 市场上的已有竞品以及我们准备研究的竞品；

注意,这个部分并不是整个调研报告的核心部分,我们应当很快地略过这个部分,这个部分的核心目标就是解释为什么我们找了XXX产品作为我们的竞品调研对象.切忌把调研报告写成**行业分析报告**.
